## NFS
```
Signifie Network File System, soit Système de fichier en réseau. NFS est un protocole permettant l'échange de fichiers par le réseau. Ce protocole s'appuie sur la couche TCP/IP de la pile de protocoles réseau.
```

## CIFS
```
Common Internet File System. Ancienne dénomination du protocole SMB. Il est encore utilisé par abus de langage pour le partage de fichiers du protocole SMB. Il était présent sous Windows NT et a disparu avec l'arrivée de Vista en 2006
```

## IP
```
Signifie Internet Protocol. IP est un ensemble de règles sur l'échange de données par paquets (paquet IP). Les fonctions principale d'IP sont l'identiication unique sur un réseau et le routage des paquets dans le ou les réseaux.
```

## Identification
```
Ensemble d'actions permettant à une entité de se faire reconnaître d'un système par un élément dont on l’a doté. En résumé, s’identifier (les actions d'identification) c’est communiquer son identité.
```

## ACL
```
Access Control List. Liste de contrôle d'accès. Est un système permettant de gérer les droits d'accès aux fichiers plus ou moins détaillé en fonction des systèmes d'exploitation (Unix vs Windows). Est également une règle d'identification d'un flux réseau.
```

## Internet
```
Internet est un ensemble de réseaux publics interconnectés entre eux. Tout service réseau peut être présent sur Internet. Les plus courants sont le partage de fichier, le DNS, le Web ou encore la transmission de vidéos.
```

## UTC
```
Universal Time Coordinated. Temps Universel Coordonné. est une échelle de temps adoptée par la majorité des pays du globe. L'UTC est un compromis entre le temps atomique international et le temps universel.
```

## Bug
```
Défaut de fonctionnement due à une erreur volontaire ou involontaire de conception ou de mise en œuvre. Il peut être physique ou logiciel et peut avoir des répercutions financières importante surtout s'il est découvert en phase d'utilisation.
```

## Port
```
Utilisé par les protocoles TCP ou UDP pour identifier le service applicatif auquel appartient un paquet de donnée. La représentation décimal d'un port est utilisée suivi du protocol comme suit : 80/tcp pour HTTP, 53/udp pour DNS.
```

## Authentification
```
A pour but de vérifier une identité. Il s'agit de l'ensemble des actions permettant de s'assurer qu'une identité est authentique (l'entité apporte la preuve de son identité). De fait, l’authentification doit être précédée d'une identification.
```

## Administration
```
L'administration d’un système d’information (SI) englobe les opérations de configuration et de gestion des ressources telles que l'installation, la gestion des configurations, la maintenance, les évolutions, la supervision et la gestion de la sécurité.
```

## RFC
```
Request for Comments. Document de spécification d'un protocole géré par des groupes de travail de L'IETF. Ces documents permettent de décrire les fonctionnements théoriques des protocoles et de décrire les exigences pour la conception d'outils et de logiciels.
```

## URL
```
Uniform Ressource Locator : chaine de caractères permettant d'identifier une ressource de façon uniforme sur le Web par son emplacement. Une URL précise également le protocole qu'il faut utiliser pour la récupérer.
```

## FAI
```
Fournisseur d'accès à Internet. Société (souvent privée) permettant d'accéder à Internet par le moyen d'une "box". Dans de nombreux cas, les FAI sont également des fournisseurs de services avec la téléphonie fixe ou la télévision.
```

## IETF
```
Internet Engineering Task Force : organisme à but non lucratif régulant les protocoles utilisés sur internet et plus globalement sur les réseaux
```

## Nagivateur Web
```
Outil logiciel permettant d'afficher les ressources issues du Web notamment un contenu en HTML, la forme des pages CSS et des animations en JAVASCRIPT. Le navigateur web est devenu le principal outil d'accès aux applicatifs.
```

## GNS3
```
Outils de simulation de réseau permettant l'intégration de machines virtuelles en isolant les flux sur votre machine. Cet outil permet de reproduire des réseaux complexe et réaliste afin de créer des prototypes afin de pouvoir tester des configurations en sécurité.
```

## SMB
```
Server Message Block. Est un protocole permettant le partage de ressources en environnement Microsoft Windows. Outre le partage de fichiers, il permet également le partage d'imprimantes. Il était aussi appelé CIFS.
```

## SI
```
Système d'Information : regroupe les ressources liées au traitement des données nécessaire à l'atteinte d'objectifs métier. Les ressources pouvant être techniques (serveurs,  ...), humaines, organisationnelles (procédures) et physiques.
```

## RTC
```
Real Time Clock. est une horloge permettant un décompte très précis du temps. Ce circuit électronique se base sur un quartz piézoélectrique. Il permet de fournir une source de temps au système d'exploitation.
```

## WiFi
```
Wireless Fidelity (à l'image de Hi-Fi). Le Wifi est une technologie permettant une communication sans fils suivant le standard IEEE 802.11. A ce jour le nom Wifi est entrée dans les usages et la version 7 est sur le point de sortir.
```

## DNS
```
Domain Name System, système de nom de domaine. Ce protocole est peut être le plus important du web. Il permet de faire correspondre une adresse IP à un nom. Il repose sur une hiérarchie de noms de domaines : www.apple.com.
```

## WEB
```
Forme raccourcie de World Wide Web. Est la "Toile mondiale" ou la "Toile". Il s'agit d'un système hypertexte public fonctionnant sur Internet. Le Web permet le partage, l'échange et la création de contenus publics.
```

## FQDN
```
Fully Qualified Domain Name : Nom pleinement qualifié d'une ressource sur un réseau. Ce nom identifie un équipement (généralement une ressource) sur Internet à l'aide de la résolution de nom faite via le protocole DNS.
```

## OS
```
Operating system. Système d'exploitation. Il s'agit du premier logiciel installé sur un ordinateur permettant de gérer l'exécution de toutes les autres applications ainsi que les accès aux périphériques matériels (imprimantes, écrans, ...)
```

## Adresse IPv4
```
Une adresse IP est une adresse logique permettant d'identifier un équipement mis en réseau. La version majoritairement utilisée est la 5. L'IPv4 se caractérise par l'utilisation de 4 octets en décimal pointé (w.x.y.z).
```

## NFS
```
Network File System. Système de fichier en réseau. Protocole client/serveur permettant d'accéder à des fichiers au travers d'un réseau comme s'ils étaient présent physiquement sur le client (alors qu'ils sont sur le serveur).
```

## Broadcast
```
Diffusion générale (broadcast en anglais). Sur un réseau, certains protocoles utilisent le broadcast afin de se faire connaitre de tous les hôtes connecté. Ainsi le destinataire est assurément informé de la requête ou de l'information.
```

## Service réseau
```
est une application accessible sur un réseau et donc utilisable par de nombreux clients. Les services réseaux sont généralement portés par des serveurs à destination de clients. Ils opèrent au niveau de la couche application (OSI).
```

## Modèle OSI
```
est un modèle théorique permettant de simplifier la gestion des communications au sein d'un réseau. Constitué de 7 couches dans sa version historique (la plus utilisée), il explicite l'empilement de protocole nécessaire au réseau.
```

## AD
```
Active Directory. LDAP étendu par Microsoft afin de simplifier la gestion de réseaux d'ordinateurs sous Windows. Est annuaire ainsi qu'un ensemble de services permettant l'identification, l'authentification et les autorisations d'accès à des ressources d'entreprises.
```

## DNSSEC
```
Extension de sécurité du protocole DNS. Permet de signer les enregistrements à l'aide d'une paire de clé (publique et privée) se basant sur des algorithmes de chiffrement et d'intégrité permettant d'en garantir l'authentification et l'intégrité.
```
