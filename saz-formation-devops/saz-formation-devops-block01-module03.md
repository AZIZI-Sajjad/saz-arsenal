## Faille de sécurité
```
Vulnérabilité dans un système informatique pouvant être exploitée par un attaquant.
```

## Ansible
```
Outil d’automatisation de configuration système et de déploiement applicatif sans agent.
```

## CI/CD
```
Ensemble d’automatisations pour intégrer, tester et déployer le code de manière continue.
```

## Pipeline
```
Chaîne automatisée d’étapes (build, test, déploiement) exécutée à chaque modification de code.
```

## Agilité
```
Approche de gestion de projet favorisant l’adaptation rapide au changement par des cycles courts et itératifs.
```

## Git
```
Système de gestion de version décentralisé permettant de suivre l’évolution du code source
```

## Rétrospective
```
Réunion en fin de sprint pour identifier les points à améliorer dans la collaboration d’équipe.
```

## Sprint Review
```
Réunion de présentation des développements terminés aux parties prenantes pour obtenir des retours.
```

## Merge Request (MR)
```
Proposition de fusion d’une branche de développement dans la branche principale.
```

## Product Owner (PO)
```
Personne responsable de la vision produit et de la priorisation du backlog
```

## Terraform
```
Outil d’IaC permettant de décrire et provisionner des infrastructures cloud de façon déclarative.
```

## Dette technique
```
Accumulation de compromis techniques qui nuisent à la maintenabilité du code sur le long terme.
```

## User Story
```
Description d’un besoin fonctionnel du point de vue de l’utilisateur.
```

## Pair programming
```
Pratique de développement à deux sur un même poste, favorisant la qualité du code.
```

## Observabilité
```
Capacité à comprendre l’état interne d’un système à partir de ses sorties (logs, métriques, traces).
```

## GitLab
```
Plateforme DevOps intégrant Git, CI/CD, gestion de projet et sécurité.
```

## Pod
```
Plus petite unité de déploiement dans Kubernetes, contenant un ou plusieurs conteneurs.
```

## Log
```
Enregistrement textuel d’événements ou d’erreurs générés par un système.
```

## TDD (Test-Driven Development)
```
Technique de développement qui consiste à écrire les tests avant le code.
```

## Scrum
```
Méthodologie agile structurée autour de rôles, rituels et artefacts pour gérer le développement logiciel.
```

## Infrastructure as Code (IaC)
```
Pratique qui consiste à gérer l’infrastructure informatique via du code versionné
```

## Backlog
```
Liste ordonnée et évolutive des tâches, fonctionnalités ou corrections à traiter.
```

## Monitoring
```
Surveillance en temps réel des performances, de la disponibilité et des incidents d’un système.
```

## Daily Stand-up
```
Réunion quotidienne courte pour synchroniser les membres de l’équipe et identifier les blocages
```

## Kubernetes
```
Orchestrateur de conteneurs permettant de déployer et gérer automatiquement des applications à l’échelle.
```

## Scalabilité
```
Capacité d’un système à s’adapter à une augmentation de la charge sans dégradation des performances.
```

## DevSecOps
```
Extension de DevOps intégrant la sécurité dès les premières étapes du développement.
```

## Branch
```
Version parallèle du code, utilisée pour développer une fonctionnalité sans affecter le code principal.
```

## Conteneur
```
Environnement léger et isolé contenant tout ce qu’il faut pour exécuter une application.
```

## Sprint
```
Période de développement courte et fixe (1 à 4 semaines) durant laquelle une équipe réalise des fonctionnalités priorisées
```

## Docker
```
Plateforme de conteneurisation permettant d’exécuter des applications de manière isolée et portable.
```

## Alerting
```
Déclenchement automatique de notifications en cas d’anomalie détectée dans un système
```

## Scrum Master
```
Facilitateur garant de l’application de la méthode Scrum et du bon déroulement des rituels agiles.
```

## Refactoring
```
Réécriture du code pour en améliorer la structure sans changer son comportement.
```

## DevOps
```
Culture visant à rapprocher les développeurs (Dev) et les opérations (Ops) pour livrer plus vite et plus fiablement
```
