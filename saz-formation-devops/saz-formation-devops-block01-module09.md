## elif 
```
Une instruction qui suit un "if" et exécute un bloc de code si la condition précédente est fausse et que la nouvelle condition est vraie.
```

## stdin (entrée standard)
```
Un flux de données utilisé pour fournir des entrées à un programme ou à une commande. Par défaut, il est connecté au clavier.
```

## Shell 
```
Interface utilisateur permettant d'interagir avec le système d'exploitation à l'aide de commandes textuelles.
```

## Exit code (code de sortie) 
```
Un nombre renvoyé par une commande ou un script pour indiquer son statut de réussite ou d'échec.
```

## Shebang (#!)
```
Les deux premiers caractères d'un script indiquant au système d'exploitation quel interpréteur utiliser pour exécuter le fichier.
```

## Terminal 
```
Un programme qui émule un terminal physique et permet à l'utilisateur d'interagir avec le système via une interface en ligne de commande.
```

## Script 
```
Un fichier contenant une série de commandes qui sont exécutées dans l'ordre.
```

## wildcards 
```
Des caractères spéciaux (comme *, ? et []) utilisés pour effectuer des recherches de fichiers ou de dossiers basées sur des motifs.
```

## else 
```
Une instruction qui suit un "if" ou "elif" et exécute un bloc de code si toutes les conditions précédentes sont fausses.
```

## esac 
```
La fin d'une structure "case".
```

## chown 
```
Une commande pour changer le propriétaire d'un fichier ou d'un répertoire.
```

## sed 
```
Un éditeur de flux (Stream EDitor) qui permet de réaliser des opérations de transformation de texte, comme la substitution, l'insertion ou la suppression de chaînes de caractères.
```

## if 
```
Une instruction qui exécute un bloc de code si une condition est vraie.
```

## Fonction 
```
Un bloc de code nommé qui peut être appelé avec des arguments pour effectuer une tâche spécifique.
```

## Condition 
```
Une expression qui est évaluée comme vraie ou fausse et qui détermine le déroulement d'un script.
```

## while 
```
Une boucle qui répète un bloc de code tant qu'une condition est vraie.
```

## case 
```
Une structure de contrôle qui permet de tester une variable contre plusieurs valeurs possibles.
```

## Bash (Bourne Again SHell)
```
Un interpréteur de commandes Unix et le shell par défaut de la plupart des distributions Linux.
```

## Variable
```
Un élément de stockage de données dans un script qui peut être modifié pendant l'exécution du programme.
```

## Pipe (|)
```
Un opérateur qui permet de rediriger la sortie d'une commande vers l'entrée d'une autre commande.
```

## chmod 
```
Une commande pour modifier les permissions d'accès d'un fichier ou d'un répertoire.
```

## Redirection 
```
Une technique permettant de changer la source d'entrée ou de sortie d'une commande.
```

## Argument 
```
Une valeur passée à une fonction ou un script lors de son appel.
```

## stdout (sortie standard) 
```
Un flux de données utilisé par un programme ou une commande pour afficher des informations à l'utilisateur. Par défaut, il est connecté à l'écran.
```

## export 
```
Une commande utilisée pour rendre une variable d'environnement accessible aux sous-processus d'un script.
```

## Echo 
```
Une commande utilisée pour afficher du texte à l'écran ou dans un fichier.
```

## Boucle 
```
Une structure de contrôle qui répète un bloc de code un certain nombre de fois ou jusqu'à ce qu'une condition soit remplie.
```

## for 
```
Une boucle qui répète un bloc de code pour chaque élément d'une séquence.
```

## grep 
```
Une commande qui recherche des chaînes de caractères dans des fichiers ou dans la sortie d'autres command
```
